# 指数周线行情

## 接口信息

| 项目 | 内容 |
| --- | --- |
| 接口名称 | `index_weekly` |
| 描述 | 获取指数周线行情 |
| 调取限量 | 单次最多返回 1,000 行记录，可分批获取，历史总量不限 |
| 积分要求 | 至少需要 600 积分；积分越多，可用调取频次越高 |
| 积分说明 | [Tushare 积分获取办法](https://tushare.pro/document/1?doc_id=13) |

## 输入参数

| 名称 | 类型 | 必选 | 描述 |
| --- | --- | --- | --- |
| `ts_code` | `str` | 否 | TS 指数代码 |
| `trade_date` | `str` | 否 | 交易日期，格式为 `YYYYMMDD` |
| `start_date` | `str` | 否 | 开始日期，格式为 `YYYYMMDD` |
| `end_date` | `str` | 否 | 结束日期，格式为 `YYYYMMDD` |

## 输出参数

| 名称 | 类型 | 默认显示 | 描述 |
| --- | --- | --- | --- |
| `ts_code` | `str` | 是 | TS 指数代码 |
| `trade_date` | `str` | 是 | 交易日 |
| `close` | `float` | 是 | 收盘点位 |
| `open` | `float` | 是 | 开盘点位 |
| `high` | `float` | 是 | 最高点位 |
| `low` | `float` | 是 | 最低点位 |
| `pre_close` | `float` | 是 | 上一周期收盘点位 |
| `change` | `float` | 是 | 涨跌点位 |
| `pct_chg` | `float` | 是 | 涨跌幅，单位为 `%` |
| `vol` | `float` | 是 | 成交量，单位为手 |
| `amount` | `float` | 是 | 成交额，单位为千元 |

## 接口示例

```python
import tushare as ts

pro = ts.pro_api()

# 获取上证指数指定日期范围的周线行情
df = pro.index_weekly(
    ts_code="000001.SH",
    start_date="20180101",
    end_date="20190329",
    fields="ts_code,trade_date,open,high,low,close,vol,amount",
)

# 获取指定交易日的全部可用指数周线行情
df = pro.index_weekly(
    trade_date="20190329",
    fields="ts_code,trade_date,open,high,low,close,vol,amount",
)
```

## Demo 阶段 CSV 存储约定

- 建议路径：`data/raw/index_weekly/{ts_code}.csv`
- 文件粒度：每个 ETF 跟踪指数一个文件，保存该指数的完整周线历史
- 唯一键：`ts_code + trade_date`
- 日期字段保留为 `YYYYMMDD` 字符串
- 点位、成交量及成交额保留 Tushare 原始单位与精度
- 不将缺失的成交量或成交额填充为 `0`
- 写入前与已有历史合并并按唯一键去重，随后原子覆盖该指数的 CSV

## 同步与历史回补建议

- 周线数据一般在每周最后一个交易日收盘后形成，不应假设 `trade_date` 必然是星期五。
- 日常同步可在交易周结束后的晚间执行，并在下一交易日再次回补最近数周。
- 按 `trade_date` 获取全市场数据可能触及单次 1,000 行限制；应优先使用目标指数列表逐个按 `ts_code` 获取。
- 单个指数长期历史超过 1,000 周时，应拆分日期窗口请求，并检查每批返回行数是否达到上限。
- 历史回补可按年度或五年区间分批，合并后以 `ts_code + trade_date` 去重。
- 可通过 `index_basic.ts_code = index_weekly.ts_code` 关联指数基础信息与周线行情。
